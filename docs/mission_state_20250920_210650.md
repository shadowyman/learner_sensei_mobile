## Key Architectural Insights
- `ui.ts` caches key DOM elements (including `sendButton`) at module scope, so any new control beside the send button must either reuse existing container elements or add new cached references to avoid repeated DOM queries.
- Submission flow is centralized in `handleUserInput`; the form submission triggers all downstream model updates, and `showLoading` gates interaction by disabling both textarea and send button—additional controls should respect this loading state.
- Styling and layout for the input row are tightly controlled in `index.css` via `.chat-input-form` and `.chat-input-send-button`; spacing is currently `gap: 12px` with the send button sized to 35px, implying new sibling elements will require deliberate spacing and possibly responsive adjustments.
