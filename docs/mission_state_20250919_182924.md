# Mission State Checkpoint 2025-09-19 18:29:24

## Analysis Scope & Entry Points
- `.weighted-segment.brand` in `index.html` launches the Sensei meditation overlay.
- `setupBrandHoverMeditationOverlay` and `updateSenseiMeditationOverlay` in `ui.ts` drive the overlay behavior.
- `updateCurriculumDisplay` in `ui.ts` exposes `window.curriculumState` for overlay consumption.
- `loadCurriculumAndGreet` and `initializeUI` in `index.tsx` bootstrap the UI pipeline and attach the brand listeners.
- `CurriculumState` shape in `curriculum.ts` supplies chunk lists and indices consumed by the overlay.

## Static Execution Trace
1. `initializeUI()` (from `loadCurriculumAndGreet` in `index.tsx`) calls `setupBrandHoverMeditationOverlay()`.
2. Brand icon mouseenter runs `setupBrandHoverMeditationOverlay`'s handler, fetching `window.curriculumState` and invoking `updateSenseiMeditationOverlay(curriculumState, true)`.
3. `updateSenseiMeditationOverlay` validates the state, rebuilds the action-item list for the current chunk, then calls `showMeditationOverlay()`.
4. Brand icon mouseleave calls `updateSenseiMeditationOverlay(null, false)`, which defers to `hideMeditationOverlay()` to dismiss the overlay.

## Key Architectural Insights
- Overlay rendering currently only iterates the active `curriculumState.currentTeachingChunkIndex` and has no header or navigation controls, explaining the missing "Chunk x/y" output and lack of chunk switching.
- Historical docs reference `window.switchToChunk`, but no such function exists in the present codebase, implying the prior navigation pathway was removed.
- `window.curriculumState` is the sole state conduit for the overlay; any curriculum update relies on `updateCurriculumDisplay` being invoked to refresh it.
- Animation hooks depend on the global `anime` library; without it the overlay falls back to simple CSS toggling.
