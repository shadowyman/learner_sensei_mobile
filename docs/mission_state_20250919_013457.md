# Mission State Checkpoint 2025-09-19 01:34:57

## Analysis Scope & Entry Points
- `updateSenseiMeditationOverlay` (`ui.ts`) – overlay UI injecting the new checkbox control.
- `window.switchToChunk` / `generateNextSenseiResponse` (`index.tsx`) – current chunk reset + learner model orchestration.
- `getAnalysisFromGemini` (`geminiService.ts`) – supplies `key_content_point_assessment` for comparisons.
- `updateLearnerModel` (`adaptiveEngine.ts`) and helpers `findBestMatchingContentPoint`, `updateKC` – own the canonical logic for updating understanding scores, coverage sets, and KC mastery per teaching point.
- `advanceCurriculumState` (`curriculum.ts`) – consumes `coveredPointsInCurrentChunk` / `pointsToRevisitInCurrentChunk` and phase KC to decide advancement.

## Static Execution Trace
1. `handleUserInput` forwards learner replies to `generateNextSenseiResponse`.
2. `generateNextSenseiResponse` collects chunk teaching-point texts, calls `getAnalysisFromGemini`, then passes the analysis to `updateLearnerModel`.
3. `updateLearnerModel` matches each Gemini `point_id` to a chunk teaching point, high-water-marking `learnerModel.contentPointsCoverage[point]`, awarding KC deltas `(effective - previous) * kcValue`, and toggling coverage/revisit sets around the 0.7 threshold.
4. `advanceCurriculumState` checks those sets plus KC mastery to advance the chunk/phase, resetting sets upon promotion.
5. Overlay renders current chunk state based on these structures; manual overrides must mutate the same data to stay consistent.

## Architectural Insights
- Understanding is authoritatively stored in `learnerModel.contentPointsCoverage[point].understanding_score`; manual toggles must set this to `1.0` (or `0.0`) to emulate Gemini granting/revoking mastery.
- KC awards rely on teaching-point `kcValue` and delta calculations; we must compute `(1 - previousScore)` exactly to top off mastery and ensure `awardedKcForPhasePoints` mirrors the awarded set.
- Advancement logic depends on `coveredPointsInCurrentChunk`/`pointsToRevisitInCurrentChunk`; we must sync these sets with manual score changes so curriculum stepping works identically to organic progression.
- Phase resets (`ensureTeachingPlanExists`, concept navigation, new phase initialization) already wipe the relevant structures; manual overrides should respect those flows without introducing new state stores.
