## Key Architectural Insights
- Chunk navigation bypasses learner analysis yet still routes through the full Sensei response pipeline, leveraging persistent chat history.
- The generated system prompt always appends `User: <currentUserInput>` even when navigation supplies an empty string, giving the model no explicit signal that the "user" took no action.
- "Let's Check" sections are mandated by curriculum focus whenever not in a MUST_OBEY turn; chunk resets only clear learner coverage, not the LLM conversation state.
- The meditation overlay relies on global `curriculumState`/`learnerModel` side effects, so chunk switching alters deep state before the next LLM call.
