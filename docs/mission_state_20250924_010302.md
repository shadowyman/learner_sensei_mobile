## Dependency & Side-Effect Summary
- `ModuleSelectionHandler.handlePhaseSelection`: depends on curriculum state, notepad, `displayMessage`, `streamModuleIntroduction`, `processMermaidBlocks`; mutates DOM (phase selection bubble removal), updates learner model, and sets reload metadata on the message bubble.
- `displayMessage`: depends on DOM APIs, `renderIcons`, `mermaidManager`, highlight.js; constructs/updates message bubble DOM nodes and registers reload button listener wiring `window.handleReloadSenseiMessage`.
- `streamModuleIntroduction`: depends on `chat.sendMessageStream`, `updateMessageStream`; streams Gemini output and forwards chunks to UI without final bubble normalization.
- `updateMessageStream`: depends on DOM, marked, highlight.js; mutates message bubble content while maintaining streaming map state.
- `processMermaidBlocks`: depends on DOM, `mermaidManager`, `runMermaidRecovery`; renders or repairs diagrams post-display.

## Key Architectural Insights
- Initial module introduction messages are displayed via a streaming-first path that never re-invokes `displayMessage` with `isLoading: false`, so reload button logic in `ui.ts` never executes for that bubble.
- Reload presentation relies on both `message.isReloadable` metadata and DOM dataset flags; only `displayMessage` appends the actual button.
