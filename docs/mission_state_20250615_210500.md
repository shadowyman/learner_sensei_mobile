## Key Architectural Insights
- IntroIllustrate dual-pass requirements are injected exclusively via `buildPrimaryActionInstruction` when the phase is IntroIllustrate and the primary action resolves to `Teach New Content (from current chunk)`.
- Automatic advancement relies on `advanceCurriculumState` to reset per-chunk coverage; manual chunk switching also clears `activeConsolidationState` and learner KC awards.
- Dynamic system instructions combine curriculum focus, profiler directives, and optional navigation context before streaming, so missing structural sections indicates an upstream focus classification issue rather than LLM defiance alone.
