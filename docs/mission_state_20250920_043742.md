# Mission State Checkpoint — Status Panel Three-Line Layout

## Entry Point & Scope
- Entry point: `updateCurriculumDisplay` in `ui.ts`, which fills the header status text.
- Scope: `ui.ts` (status text logic, phase naming), `index.html` (header markup for status segment), `index.css` (status segment typography, colors, spacing).
- Auxiliary helpers: `getPhaseDisplayName` in `ui.ts` for phase naming conventions.

## Static Execution Trace
1. Application state updates trigger `updateCurriculumDisplay(curriculumItem, currentPhase, ...)`.
2. `updateCurriculumDisplay` computes topic strings based on module, concept, and phase data, then updates `curriculum-status-topic` text and `status-label` content.
3. CSS in `index.css` formats `.status-label` and `.status-text` within `.weighted-segment.status`.
4. Header renders per `index.html`, showing `.status-label` and `#curriculum-status-topic` within `.status-content`.

## Dependency & Side-Effect Table
| Function / Resource | Dependencies | Side Effects |
| --- | --- | --- |
| `updateCurriculumDisplay` (`ui.ts`) | `getPhaseDisplayName`, DOM nodes (`curriculum-status-topic`, `status-segment`), `learnerModel` for KC updates | Mutates status label text, updates KC progress bar, toggles shimmer animation.
| `getPhaseDisplayName` (`ui.ts`) | Phase enum values | Returns user-facing phase names, no side effects.
| `.status-content` styles (`index.css`) | CSS variables for colors, font sizing | Determines layout, line height, color palette of header status panel.
| Header markup (`index.html`) | Static structure for `.weighted-segment.status` | Provides DOM elements targeted by JS and CSS.

## Key Architectural Insights
- Status panel text is updated centrally in `updateCurriculumDisplay`; restructuring display across multiple lines requires adjustments in this function only.
- CSS currently assumes a single line for `status-text`; multi-line layout must fit within `--layout-header-height` without changing segment height.
- The `status-label` is static in HTML; abbreviating it or recoloring may be done via DOM manipulation or direct HTML edit.
