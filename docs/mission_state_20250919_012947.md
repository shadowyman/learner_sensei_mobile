## Analysis Scope & Entry Points
- `index.tsx:handleUserInput` and `generateNextSenseiResponse` – orchestrate user turns and invoke learner analysis.
- `geminiService.ts:getAnalysisFromGemini` – issues the comprehensive analysis prompt capturing chunk action items.
- `adaptiveEngine.ts:updateLearnerModel` (plus helpers `findBestMatchingContentPoint`, `updateKC`) – interprets Gemini feedback, awards understanding scores, and updates KC mastery.
- `curriculumState` tracking sets (`coveredPointsInCurrentChunk`, `pointsToRevisitInCurrentChunk`) and `learnerModel.contentPointsCoverage` – store per-action-item progress.

## Static Execution Trace
1. **`handleUserInput`** packages the learner’s reply, displays it, and calls `generateNextSenseiResponse`.
2. **`generateNextSenseiResponse`** derives `expectedContentPointTextsForCurrentChunk` (all teaching-point texts in the active chunk) and invokes **`getAnalysisFromGemini`** with the user text, last Sensei response, current phase KC id, and expected chunk points.
3. **`getAnalysisFromGemini`** calls Gemini with the comprehensive-analysis prompt and returns structured `key_content_point_assessment` entries (each includes `point_id`, `coverage`, `understanding_score`).
4. **`generateNextSenseiResponse`** forwards the analysis plus the expected point list to **`updateLearnerModel`**.
5. **`updateLearnerModel`**
   - Normalizes the assessment list via `findBestMatchingContentPoint` (string matching between assessment `point_id` and actual teaching point text).
   - Applies high-water marking to `learnerModel.contentPointsCoverage[point]`, promoting `understanding_score` when the new score exceeds the previous one.
   - For each improvement, computes the KC delta `(effectiveScore - previousScore) * teachingPoint.kcValue` and calls `updateKC` to increment the current phase KC (capped at 1.0) while tracking awarded points in `learnerModel.awardedKcForPhasePoints`.
   - Updates `curriculumState.coveredPointsInCurrentChunk` / `pointsToRevisitInCurrentChunk` depending on whether the effective score crosses the 0.7 mastery threshold.
6. Updated mastery/coverage feeds back into UI refresh (KC progress bar, overlay states) and subsequent curriculum progression checks.

## Architectural Insights
- Understanding is tracked per teaching point via `contentPointsCoverage[point.text].understanding_score`, maintained as a high-water mark to prevent regressions from lowering previously earned credit.
- KC awards tie directly to teaching point metadata (`TeachingPoint.kcValue`) and only accrue when the effective understanding score increases; this ensures idempotency across repeated confirmations.
- Curriculum advancement relies on the coverage/revisit sets, so any manual override must keep them in sync with `contentPointsCoverage` and KC totals to avoid progression mismatches.
- Gemini prompt includes the expected chunk point texts, enabling robust matching even when the model returns slight wording variations (handled by fuzzy matching).
