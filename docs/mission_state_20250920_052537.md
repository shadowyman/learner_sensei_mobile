# Mission State Checkpoint — SVG Navigation Icons

## Entry Point & Scope
- Entry point: Header navigation buttons inside `index.html` (`concept-nav-prev/next`, `chunk-nav-prev/next`).
- Scope: `index.html` (button markup), `index.css` (navigation styling), potential shared SVG assets directory (tbd) or inline SVG definitions.
- Supporting logic: `ui.ts` (`updateConceptNavigationArrowsUI`), `index.tsx` (`handleConceptNavigation`, `handleChunkNavigation`) for button behavior; no changes expected but must confirm compatibility with new SVG structure.

## Static Execution Trace
1. `updateCurriculumDisplay` updates header text and calls `updateConceptNavigationArrowsUI` (`ui.ts`).
2. `updateConceptNavigationArrowsUI` toggles visibility/disabled state for chunk and concept buttons.
3. In the DOM, buttons currently render arrow glyph spans; click listeners (from `index.tsx`) invoke `handleConceptNavigation` / `handleChunkNavigation` and eventually `switchToChunk`.
4. CSS in `index.css` styles `.nav-group`, `.concept-nav-arrow`, `.chunk-nav-arrow`, and `.arrow-icon` spans.

## Dependency & Side-Effect Table
| Component | Dependencies | Side Effects |
| --- | --- | --- |
| Header buttons (`index.html`) | CSS classes, listeners registered in `index.tsx` | Render icons, expose navigation controls. |
| `.concept-nav-arrow` / `.chunk-nav-arrow` (`index.css`) | DOM structure under buttons | Defines layout, hover states, icon sizing. |
| `updateConceptNavigationArrowsUI` (`ui.ts`) | DOM references to buttons, curriculum state | Shows/hides/ disables buttons; no rendering logic. |
| `handleConceptNavigation` / `handleChunkNavigation` (`index.tsx`) | Buttons’ event listeners | Moves curriculum state, triggers re-render; unaffected by icon type. |

## Key Architectural Insights
- Buttons currently rely on inline spans with text glyphs; replacing with SVGs requires ensuring the SVG dimensions integrate with existing button size (26x26px) and hover styles.
- Since `ui.ts`/`index.tsx` only manipulate buttons, swapping inner content to SVGs should not impact logic, but we must preserve `.arrow-icon` hooks or update CSS accordingly.
- Consider centralizing SVG definitions (e.g., inline `<svg>` inside buttons or referencing `<symbol>` via `<use>`). Inline SVG ensures no asset pipeline changes.
