2. `generateNextSenseiResponse()` orchestrates:
   - `ensureTeachingPlanExists()` → `generateTeachingPlanForPhase()` (LLM) if chunk missing.
   - `calculateFocusStrategy()` → `calculateFocusPoints()` to select teaching items.
   - `pedagogicalProfiler.getDirective()` → `generateDirectiveFromMetaPrompt()` (LLM) for teaching style.
   - `getCurriculumFocusInstruction()` → `buildPrimaryActionInstruction()` → chunk templates in `prompts.ts`.
   - Dynamic context split:
     - Socratic: `buildSocraticExecutionInstruction()` / `buildSocraticInitialInstruction()`.
     - Non-Socratic: `buildSenseiDynamicSystemInstruction()` → `MAIN_SENSEI_RESPONSE_SYSTEM_INSTRUCTION_TEMPLATE_FUNCTION()`.
3. `streamMainSenseiResponse()` sends `[dynamicContext + user input]` to Gemini via `Chat.sendMessageStream()`, updating UI as tokens arrive.

## Dependency & Side-Effect Highlights
- `generateNextSenseiResponse()` depends on curriculum state, learner model, profiler directive, focus calculation; side effects include LLM calls, state mutation, UI rendering, mermaid processing.
- `calculateFocusStrategy()` relies on `calculateFocusPoints()` and current chunk metadata; pure data shaping.
- `calculateFocusPoints()` examines `curriculumState` collections; mutates nothing but drives action selection.
- `getCurriculumFocusInstruction()` composes structured directives using templates; deterministic string build.
- `buildPrimaryActionInstruction()` selects template (`TEACH_NEW_CONTENT_CHUNK_PROMPT_TEMPLATE`, etc.); injects focus bullet lists.
- `TEACH_NEW_CONTENT_CHUNK_PROMPT_TEMPLATE` et al. embed depth/quality requirements but no explicit length target.
- `buildSenseiDynamicSystemInstruction()` parses MUST_OBEY flags and wraps guidance via `MAIN_SENSEI_RESPONSE_SYSTEM_INSTRUCTION_TEMPLATE_FUNCTION()`.
- `MAIN_SENSEI_RESPONSE_SYSTEM_INSTRUCTION_TEMPLATE_FUNCTION()` builds the core checklist the tutor must follow; current copy demands depth but not quantitative length.
- `buildSocraticExecutionInstruction()`/`buildSocraticInitialInstruction()` craft per-turn instructions from teaching plan metadata (expected turns, completion triggers); also lacks explicit token guidance besides qualitative rules.
- `pedagogicalProfiler.getDirective()` delegates to Gemini (Flash) with meta-prompt emphasizing strategy; output may alter tone/structure but not guaranteed to expand length.
- `streamMainSenseiResponse()` merges dynamic context with user text and streams model output; primary side effects are network call, UI updates.

## Key Architectural Insights
- Response verbosity is primarily governed by the curriculum focus templates and the main system instruction checklist. None specify token or paragraph counts; they only exhort "immense depth" qualitatively.
- Socratic mode relies on the teaching plan's `interactionGuidance.expectedTurns`; doubling message length may conflict with multi-turn expectations unless prompts are carefully aligned.
- Model configs (`MAIN_SENSEI_RESPONSE_CHAT_MODEL_CONFIG`) do not impose `maxOutputTokens`, so output truncation is unlikely; length control must come from prompt directives.
- Pedagogical directives can bias style but change each turn; durable length adjustments require edits to deterministic prompt scaffolding (`prompts.ts`, `curriculum.ts`).
