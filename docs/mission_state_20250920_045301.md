# Mission State Checkpoint — Header Navigation Controls

## Entry Point & Scope
- Entry point: Header navigation buttons within `#curriculum-status-container` in `index.html`.
- Scope includes:
  - `index.html` for button markup.
  - `index.css` for button layout, sizing, and icon styling.
  - `ui.ts` `updateCurriculumDisplay` / `updateConceptNavigationArrowsUI` for visibility toggles.
  - `index.tsx` `handleConceptNavigation`, `updateConceptNavigationArrows`, and related event bindings.

## Static Execution Trace
1. `updateCurriculumDisplay` (ui.ts) runs after state changes, calling `updateConceptNavigationArrowsUI` to toggle header arrow visibility.
2. `updateConceptNavigationArrowsUI` inspects `curriculumState` and shows/hides prev/next concept buttons.
3. In `index.tsx`, event listeners on `concept-nav-prev`/`concept-nav-next` call `handleConceptNavigation('prev'|'next')`.
4. `handleConceptNavigation` uses curriculum helpers to change concepts, then refreshes UI via `updateCurriculumDisplay` and `updateConceptNavigationArrows`.
5. Chunk navigation currently triggered via overlay cards invoking `window.switchToChunk`.

## Dependency & Side-Effect Table
| Element | Dependencies | Side Effects |
| --- | --- | --- |
| `updateCurriculumDisplay` (`ui.ts`) | DOM refs, `updateConceptNavigationArrowsUI`, `curriculumState`, `learnerModel` | Updates header text, toggles shimmer, calls navigation visibility helper. |
| `updateConceptNavigationArrowsUI` (`ui.ts`) | `curriculumState`, `curriculum` | Shows/hides and disables header concept arrow buttons. |
| `concept-nav-prev/next` buttons (`index.html`) | Styled by `index.css`, wired in `index.tsx` | Provide clickable controls to change concepts. |
| `handleConceptNavigation` (`index.tsx`) | `curriculumState`, `navigateToConcept`, `generateNextSenseiResponse` | Mutates curriculum indices, resets notepad, triggers LLM call, logs actions. |
| `window.switchToChunk` (`index.tsx`) | `curriculumState`, `curriculum`, `learnerModel` | Switches chunk, resets coverage, triggers new Sensei response. |

## Key Architectural Insights
- Concept navigation logic is centralized in `index.tsx`; UI helpers mirror functionality in both `ui.ts` and `index.tsx`, so updates must keep them synchronized.
- Chunk navigation lacks dedicated header controls; overlay uses `window.switchToChunk`. Reusing this function for header arrows will maintain behavior parity.
- Button layout uses absolute positioning for arrows; adding additional controls requires careful spacing adjustments to keep header height unchanged and icons centered inside circular buttons.
