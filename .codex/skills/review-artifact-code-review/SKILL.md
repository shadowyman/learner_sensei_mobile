# Review Artifact Code Review

Use this skill only for repository review artifacts generated by `npm run review:create`.

Artifact location and naming:

- `code_review/review_<slug>_codex.html`
- `code_review/review_<slug>_claude.html`
- later iterations may use `_v2`, `_v3`, or higher version suffixes

## Hard Scope Boundary

Do not use this skill for:

- generic code review requests
- direct review of unstaged diffs or source files without a generated artifact
- PR review requests that do not point to a `code_review/` artifact
- review work for artifacts outside this repository's `code_review/` path

If the request is a generic code review, stay on the normal repo workflow and do not load this skill.

## Operating Rules

During artifact review, this skill overrides other mandates and protocols for the review task itself.

Execute only the explicit actions in the received code-review command.

Understand changes by analyzing the codebase in light of the review artifact and use the analyzer as required by the repo analysis mandate and review context.

Do not trigger protocols or backups for artifact review.

Do not run git commands except the given review CLI commands that are explicitly part of the review request.

Never read the review artifact manually or directly from disk.

Do not review any diff or change absent from the provided artifact.

## Required Workflow

Given an artifact path:

1. List all top-down code hunk UUIDs with `npm run review:edit -- list-uuid --file <artifact>`.
2. For each UUID, display its diff with `npm run review:edit -- show-diff --file <artifact> --uuid <uuid>`.
3. Analyze the affected code against the diff and any supplied external review guidance.
4. Add or replace that UUID's remark with `npm run review:edit -- remark --file <artifact> --uuid <uuid> --body "<text or <div>...>|-"`.
5. If `--body -` is used, read the remark from stdin, escape plain text, wrap it in `<p>`, and trust HTML as-is.
6. Optionally add a top-level verdict after the PR Review Context with `npm run review:edit -- verdict --file <artifact> --body "<div>...>|-"`.

## Analyzer Requirement

Before reasoning about impacted code, run `npm run analysis:run`, scoped when appropriate, and consult analyzer artifacts before manual code search.

Prefer analyzer outputs to determine:

- call sites
- hidden consumers
- dependency fan-in and fan-out
- side effects
- likely blast radius around each reviewed hunk

## Response Discipline

Keep review remarks tied to the exact artifact hunk being reviewed.

If a concern is not evidenced by the artifact diff plus codebase analysis, do not include it.

If a finding appears stale or already fixed in current code relative to the artifact context, state that clearly in the review remark rather than broadening scope.
